26// queue decrease algorithm
27// since we have the access to the maximum queue value, and it is being checked in the
28// ptrFromIndex(), we have a way to restrict the access to certain nodes for the writeToLog()
29// All by changing the max queue value to the value of the second-to-last node in the allocation vector
30// That way, when we wrap the index around next time, it wraps around the nodes that we want to keep
31// And completely ignores the last nodes we want to delete.
32// Though how to delete information from those delete-me nodes, since the writer-thread is the only thing
33// That still has access to those nodes. And we don't want to delete the unfilled ones
