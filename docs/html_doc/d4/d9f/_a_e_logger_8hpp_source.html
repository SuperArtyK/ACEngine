106// queue decrease algorithm
107// since we have the access to the maximum queue value, and it is being checked in the
108// ptrFromIndex(), we have a way to restrict the access to certain nodes for the writeToLog()
109// All by changing the max queue value to the value of the second-to-last node in the allocation vector
110// That way, when we wrap the index around next time, it wraps around the nodes that we want to keep
111// And completely ignores the last nodes we want to delete.
112// Though how to delete information from those delete-me nodes, since the writer-thread is the only thing
113// That still has access to those nodes. And we don't want to delete the unfilled ones
